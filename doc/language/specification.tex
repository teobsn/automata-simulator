\section{Introduction}
The Automaton Specification Language (ASL) is a domain-specific language designed to define finite automata for simulation. It supports both Deterministic Finite Automata (DFA) and Non-deterministic Finite Automata (NFA).

\clearpage
\section{File Structure}
An ASL file is divided into sections, each denoted by a header in square brackets. Sections can appear in any order, though the following order is conventional:
\begin{enumerate}
    \item \texttt{[alphabet]}
    \item \texttt{[states]}
    \item \texttt{[initial\_state]}
    \item \texttt{[accept\_states]}
    \item \texttt{[transitions]}
    \item \texttt{[settings]} (optional)
\end{enumerate}

\clearpage
\section{Sections}

\subsection{Alphabet}
The \texttt{[alphabet]} section defines the set of symbols accepted by the automaton. Symbols are space-separated and can span multiple lines.

\subsection{States}
The \texttt{[states]} section lists all possible states in the automaton. Like the alphabet, states are space-separated.

\subsubsection{State Ranges}
To define a sequence of states with a common prefix and a numerical range, you can use the range syntax: \texttt{prefix[start-end]}.
\begin{lstlisting}
q[0-15] # Expands to q0 q1 q2 ... q15
\end{lstlisting}

\subsection{Initial State}
The \texttt{[initial\_state]} section must contain exactly one state from the defined states list. This represents the starting point of the simulation.

\subsection{Accept States}
The \texttt{[accept\_states]} section lists the states that lead to an 'Accepted' result if the simulation ends in one of them. State ranges are also supported here.

\subsection{Transitions}
The \texttt{[transitions]} section defines the state transition function.

\subsubsection{Wildcards}
The asterisk \texttt{*} can be used as a wildcard for input symbols, representing all symbols in the defined alphabet.
\begin{lstlisting}
q0, * -> q1 # Transition to q1 on any symbol from the alphabet
\end{lstlisting}
In DFAs, a specific transition defined for a symbol will take precedence over a wildcard transition if it appears later in the file.

\subsection{Settings}
The \texttt{[settings]} section allows for optional behavioral modifications of the automaton. Settings are listed line-by-line or space-separated.

\subsubsection{RemainInCurrentStateOnUndefinedTransition}
When this setting is enabled, the automaton will not 'Hang' if it encounters a symbol for which no transition is defined for the current state. Instead, it will remain in its current state and continue processing the rest of the input string. This is particularly useful for building text-based games or parsers where some inputs should simply be ignored or keep the state unchanged.

\clearpage
\section{Transition Syntax}

\subsection{DFA Transitions}
For Deterministic Finite Automata, each transition must follow the strict format:
\begin{lstlisting}
source_state, symbol -> next_state
\end{lstlisting}

\subsection{NFA Transitions}
Non-deterministic Finite Automata support additional syntax for flexibility:

\textbf{Single Transition:}
\begin{lstlisting}
q0, a -> q1
\end{lstlisting}

\textbf{Multiple Next States:}
\begin{lstlisting}
q0, a -> q1, q2
\end{lstlisting}

\textbf{Multiple Input Symbols:}
\begin{lstlisting}
q0, (a, b) -> q1
\end{lstlisting}

\textbf{Wildcard Transition:}
\begin{lstlisting}
q0, * -> q1
\end{lstlisting}

\clearpage
\section{Lexical Rules}
\begin{itemize}
    \item \textbf{Comments:} Any text following a \texttt{\#} symbol on a line is ignored.
    \item \textbf{Whitespace:} Leading and trailing whitespace on lines is ignored. Empty lines are ignored.
    \item \textbf{Case Sensitivity:} Section headers are case-insensitive (\texttt{[ALPHABET]} is the same as \texttt{[alphabet]}).
\end{itemize}

\section{Example}
\begin{lstlisting}[language=bash, caption=Example DFA file]
[alphabet]
a b

[states]
s0 s1

[initial_state]
s0

[accept_states]
s1

[transitions]
s0, a -> s1
s1, b -> s1
s1, a -> s0
\end{lstlisting}

\begin{lstlisting}[language=bash, caption=Example NFA file with ranges and wildcards]
[alphabet]
0 1

[states]
q[0-2]

[initial_state]
q0

[accept_states]
q2

[transitions]
q0, * -> q0
q0, 1 -> q1
q1, (0, 1) -> q2
\end{lstlisting}
